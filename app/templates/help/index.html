    {{ _('How metadata moves through the platform') }}

    {{ _('The complete path from institutional discovery to analytics and exports.') }}

    1. {{ _('1. Resolve the institution') }}

      {{ _('The registry identifies the institution by ROR and loads any verified GRID and Ringgold identifiers available for broader discovery.') }}

    2. {{ _('2. Search ORCID') }}

      {{ _('Separate expanded searches are run for each institutional identifier. Results retain their match provenance and are deduplicated by ORCID iD.') }}

    3. {{ _('3. Preserve researcher relationships') }}

      {{ _('Verified search results are stored before profile downloads. Legacy cache relationships can be backfilled as inferred evidence and remain clearly labeled until a verified search replaces them.') }}

    4. {{ _('4. Build ORCID caches') }}

      {{ _('Public profile names, works, funding records, and institutional management status are downloaded and stored with synchronization results.') }}

    5. {{ _('5. Build canonical outputs and enrich with OpenAlex') }}

      {{ _('ORCID work records are linked to a canonical output by normalized DOI or a conservative title-and-year fallback. OpenAlex then enriches eligible outputs.') }}

    6. {{ _('6. Present and export') }}

      {{ _('Dashboards, quality reviews, institutional comparisons, and downloadable files are generated from the local caches.') }}

    {{ _('The ANID Open Access module counts the native OpenAlex oa_status categories diamond and green.     {{ _('Counts and percentages follow the active dashboard filters. Percentages use all filtered OpenAlex-enriched articles as the denominator, so Diamond and Green do not represent the full Open Access total.') }}

    {{ _('The module also ranks journals or sources by Diamond and Green article volume and citations, provides independently paginated journal and article tables, and adds server-side search and column sorting across every matching record, not only the visible page.     {{ _('For managers and administrators, the global comparison includes the same paginated rankings, annual line charts for articles and citations, and institutional statistics.     {{ _('Technical architecture') }}

    {{ _('A maintainable overview of how the application is constructed.') }}

    {{ _('Application') }}Flask 3 · Blueprints

    {{ _('Routes are separated by authentication, administration, dashboards, data management, exports, and integrations.') }}

    {{ _('Interface') }}Jinja2 · AdminLTE · Bootstrap

    {{ _('Server-rendered templates use shared layout components, Chart.js, DataTables, Font Awesome, and responsive CSS.') }}

    {{ _('Persistence') }}PostgreSQL · SQLAlchemy · Flask-Migrate

    {{ _('PostgreSQL stores operational metadata, canonical outputs, durable synchronization jobs, review decisions, and analytical caches. SQLite remains supported for isolated tests.') }}

    {{ _('External services') }}ORCID · ROR · OpenAlex

    {{ _('HTTP clients include retry handling, rate-limit awareness, normalized identifiers, and conservative matching rules.') }}

    {{ _('Exports') }}pandas · openpyxl

    {{ _('Tabular datasets are generated as CSV or Excel-compatible files from cached records.') }}

    {{ _('Localization') }}Flask-Babel
